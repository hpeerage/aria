# 자동 버전 관리 및 콜백 워크플로우

본 프로젝트의 모든 변경 사항은 추적이 가능하며, 작업 단위별로 원격 저장소에 자동으로 동기화됩니다. 문제가 발생할 경우 이전 시점으로 되돌리는 '콜백'이 가능하도록 설계되었습니다.

## 1. 버전 기록 규칙 (AI 에이전트 전용)

- **커밋 시점**: `task.md`의 세부 항목이 1개 이상 완료되거나, 독립적인 기능 단위가 완성되면 즉각 커밋을 수행합니다.
- **커밋 메시지**: `[ARIA] <feat/fix/style>: <설명>` 형식으로 기록합니다.
- **태그**: 주요 기능(마일스톤) 완성 시 `v0.x.x` 형식으로 태그를 생성합니다.

## 2. 자동 동기화 명령어

에이전트는 모든 작업 완료 후 `scripts/sync.sh`를 실행하여 원격 저장소(`origin/main`)에 푸시를 완료합니다.

// turbo
```bash
./scripts/sync.sh "[ARIA] Update: <TaskName>"
```

## 3. 콜백(롤백) 방법 (사용자 가이드)

예기치 못한 오류나 이전 디자인으로 돌아가고 싶을 때, 다음 명령어를 통해 안전하게 콜백 할 수 있습니다.

### 특정 커밋으로 돌아가기
1. `git log --oneline` 으로 되돌리고 싶은 시점의 해시값을 확인합니다.
2. 다음 명령어를 실행합니다:
   ```bash
   git reset --hard <해시값>
   git push origin main --force
   ```

### 태그 시점으로 돌아가기 (안전)
태그가 지정된 안정적인 버전으로 돌아가고 싶을 때:
```bash
git reset --hard v0.1.0  # 예시
```
